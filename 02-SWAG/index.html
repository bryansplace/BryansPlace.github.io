Remote access to HASS

Objective

Enable remote access to HomeAssistant.

Running Home assistant in a docker container, remote access can be a headache as the standard Hass io system add-ons are not available.

There is a docker package SWAG (Secure Web Application Gateway) that sets up an Nginx reverse proxy and a built-in certbot client that automates SSL server certificate generation and renewal processes (Let's Encrypt). It also contains fail2ban for intrusion prevention.

The following are the basic steps to get it installed and working.

1) Create Data Volume

Firstly, create the data volume so that you own it. SWAG will put a lot of stuff in there.

mkdir ~/volumes/swag

2) Port Forwarding

Forward ports 80 and 443 through your router to the ports 80 and 443 of the homeassistant server. No need to forward port 8123.

3) DuckDNS

Set up a Duckdns account. This is simple and fully explained on their web site. Keep a record of “your-domain” and “your-access-token”.

4) Docker-compose.yaml

Add the configuration below into your docker-compose.yaml file.

version: "3"
services:
  homeassistant:
    ...
    ...

  swag:
    image: 'ghcr.io/linuxserver/swag:arm32v7-1.32.0'
    container_name: swag
    cap_add:
      - NET_ADMIN
    environment:
  #    - DOCKER_MODS='ghcr.io/linuxserver/mods:swag-dashboard'
      - PUID=10000
      - PGID=10000
      - TZ=Europe/Paris
      - URL=50460.duckdns.org
      - VALIDATION=duckdns
      - DUCKDNSTOKEN=a5011e49-b4f1-47fb-9514-c96dc16bba71
      - SUBDOMAINS=wildcard
#      - CROWDSEC_API_KEY=c0504ad4245e5563b5f650895f874a7b
#      - CROWDSEC_LAPI_URL=http://192.168.1.111:8080
    volumes:
      - /home/bryan/volumes/swag:/config
    network_mode: host
    restart: unless-stopped

Since docker creates some files as root, you will need your PUID & GUID. Use the Unix command 'id' to find these (mine were 10000).

5) Start SWAG

Run docker-compose up swag

This will down load the swag image, create the swag volume, unpack and set up the default configuration. It takes quite some time to generate the certificates etc

If all goes well, it should end with

swag | Server ready.

Use ctrl-c to stop docker gracefully.

6) SWAG config for Homeassistant

SWAG has pre-configured Homeassistant file that first needs to be renamed: 

cd  ~/volumes/swag/nginx/proxy-confs
cp homeassistant.subdomain.conf.sample homeassistant.subdomain.conf

Since Homeassistant uses the Network mode "host", docker hasn't given your machine a name, so we need to change the two lines in this file containing set $upstream_app homeassistant; to your HA address set $upstream_app 192.168.X.XXX;

To help, this is the file (with all the comment lines removed for clarity)

server {
    listen 443 ssl;
    listen [::]:443 ssl;
    server_name homeassistant.*;
    include /config/nginx/ssl.conf;
    client_max_body_size 0;
    location / {
        include /config/nginx/proxy.conf;
        include /config/nginx/resolver.conf;
        set $upstream_app 192.168.XX.XXX;
        set $upstream_port 8123;
        set $upstream_proto http;
        proxy_pass $upstream_proto://$upstream_app:$upstream_port;
    }
        location /api {
        include /config/nginx/proxy.conf;
        include /config/nginx/resolver.conf;
        set $upstream_app 192.168.XX.XXX;
        set $upstream_port 8123;
        set $upstream_proto http;
        proxy_pass $upstream_proto://$upstream_app:$upstream_port;
    }
}

7) HA config.yaml

Add the following to you home assistant config.yaml ( /home/user/volumes/hass/configuration.yaml). Adjust for your local lan network

http:
  ip_ban_enabled: true
  login_attempts_threshold: 3
  use_x_forwarded_for: true
  trusted_proxies:
    - 192.168.X.0/24  # Local Lan

8) Finally-Test it is working

Launch homeassistant and swag

docker-compose up -d

Wait a little.

Then, use your browser to logon from your local network 192.168.X.XXX:8123 and you should get your normal home assistant login.

Finally, use your browser to logon from outside your home https://homeassistant.YOUR-SUB-DOMAIN.duckdns.org

Note: unless your router supports ’ loopback’ ( and mine didn’t) you might not be able to connect; in that case use a telephone rather than your local LAN connection.

That’s it. You have remote access to home assistant.
